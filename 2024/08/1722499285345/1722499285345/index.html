镜像认证

在k8s的master节点上查看镜像认证信息,集群的每个node节点都需要这个文件存在，文件目录均为一致。
cat /root/.docker/config.json

私有镜像地址

添加镜像地址，集群的每个node节点都需要这个文件存在，文件目录均为一致：

"insecure-registries": [ "http://10.200.40.39:8080" ]

完整如下： cat /etc/docker/daemon.json

加载：systemctl daemon-reload
重启：systemctl restart docker

kube-proxy 开启 ipvs

修改ConfigMap的kube-system/kube-proxy中的config.conf，把 mode: “” 改为mode: “ipvs” 保存退出即可


default-http-backend

解决

kubectl describe ingresses 内的 Default backend:  default-http-backend:80 (<error: endpoints "default-http-backend" not found>)

安装：kube-backend.yaml

应用：kubectl apply -f kube-backend.yaml
